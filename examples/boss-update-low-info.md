# 示例：低信息向老板汇报

用户输入：

```text
我明天要跟老板汇报一下当前项目进度，但现在有延期风险。帮我想一下这份 PPT 应该怎么讲。
```

建议输出方向：

## 1. 输入收敛

| 类型 | 内容 | 对框架的影响 |
| --- | --- | --- |
| 已确认事实 | 明天向老板汇报项目进度；存在延期风险 | 使用向上汇报 + 风险预警结构 |
| 合理假设 | 老板需要快速判断风险影响和是否需要决策 | 开头直接给状态判断，不铺背景 |
| 高影响未知 | 延期多久、影响什么目标、是否需要资源或取舍 | 结尾暂时用“需要确认的决策”占位 |

## 2. 推荐框架

1. **项目不是全面失控，真正风险集中在 X 里程碑**
   - 说明当前总体状态和唯一关键风险。

2. **已完成部分仍按计划推进，但 X 已开始偏离**
   - 展示进度事实、已完成事项和偏差。

3. **延期风险来自 2 个具体原因，而不是团队执行失速**
   - 区分资源、依赖、范围变化或外部阻塞。

4. **现在有 3 个选择，每个选择对应不同代价**
   - 保范围延期、砍范围保时间、加资源保目标。

5. **建议选择 X，并请老板今天确认 Y**
   - 明确需要老板拍板、协调或接受的事项。

## 3. 需要补充的证据

- 原计划和当前实际进度
- 延期风险影响的业务目标
- 可选方案的时间、成本和风险
- 需要老板决策或协调的具体事项
